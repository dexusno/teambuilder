# Software Development Pattern - Example Workflows

## Sprint Planning Workflow

```
Step 1: Backlog Review
  Agent: Product Manager
  Action: Present prioritized user stories, explain business value, answer questions

Step 2: Technical Breakdown
  Agents: Tech Lead + Developers
  Action: Break stories into technical tasks, identify dependencies, estimate complexity

Step 3: Capacity Planning
  Agent: Scrum Master
  Action: Calculate team capacity, account for meetings/reviews, identify risks

Step 4: Sprint Commitment
  Agents: Product Manager + Tech Lead + Team
  Action: Agree on sprint goal and stories, ensure realistic commitment

Step 5: Task Assignment
  Agents: Developers self-assign
  Action: Developers volunteer for tasks based on skills and interest
```

## Feature Development Workflow

```
Step 1: Design Review
  Agents: Tech Lead + Developers
  Action: Review technical approach, identify gotchas, agree on implementation

Step 2: Development
  Agent: Developer
  Action: Implement feature following coding standards, write unit tests

Step 3: Code Review
  Agents: Tech Lead + Peer Developer
  Action: Review code for quality, maintainability, test coverage. Provide feedback

Step 4: Testing
  Agent: QA Engineer
  Action: Execute test cases, exploratory testing, verify acceptance criteria

Step 5: Demo
  Agent: Developer
  Action: Demonstrate working feature to Product Manager and stakeholders

Step 6: Merge & Deploy
  Agent: DevOps (or Developer)
  Action: Merge to main, deploy to staging/production via CI/CD
```

**Key Learning:** Sprint ceremonies, collaborative decision-making, quality built-in, iterative delivery.
